1. 하이브리드 교차 개발 워크플로우 (Github codespace & Colab GPU)
[문제 상황] 개발 환경 내 가속 연산 장치(VRAM)의 부재
모델의 뼈대를 설계하고 단위 테스트를 진행한 GitHub Codespace 환경은 CPU 기반의 가벼운 컨테이너 환경으로, 24층의 거대한 트랜스포머 아키텍처를 학습하고 실시간 에폭별 성능 지표를 분석하기에는 하드웨어 연산 성능(VRAM 부재)이 극도로 제한되었습니다.

[해결 전략] 이원화(Two-Track) 하이브리드 워크플로우 설계
자원의 한계를 극복하기 위해 "코드 설계 및 버전 관리(Codespace)"와 "가속 훈련 및 성능 정밀 분석(Colab)" 환경을 분리하여 긴밀하게 연동하는 하이브리드 개발 루프를 구축했습니다.

Codespace (Drafting): attention.py, transformer_block.py, dataset.py 등 완벽한 모듈식 코드의 정적 틀을 설계하고 기초 구동을 검증
Google Colab (Acceleration): 깃허브 원격 저장소에 반영된 코드베이스를 코랩을 통해 빠르게 실험

Feedback: 학습 수렴 곡선과 오차 지표, 성능 분석 리포트를 확인한 뒤, 아키텍처 튜닝이 필요한 부분의 수치 및 하이퍼파라미터를 다시 Codespace의 마스터 브랜치로 가져와 영점을 맞추는 환류 체계를 안착시켰습니다.

💾 2. MLOps 관점의 데이터 볼륨 격리 및 적재 자동화
[문제 상황] 금융 데이터 확장성(Scalability)과 Git 용량 제한의 충돌
금융 시장 시계열 데이터는 백테스팅 기간이 길어지고 다루는 자산(Ticker) 수가 늘어날수록 데이터베이스 파일(.db)의 용량이 수 GB에서 수 TB까지 기하급수적으로 폭증(Scalability)하는 특성을 가집니다.

대용량 바이너리 파일을 Git 버전 관리 본진에 그대로 밀어 넣는 것은 Git LFS 오버헤드 및 저장소 대역폭 낭비를 초래하는 안티 패턴(Anti-pattern)이었습니다.

[해결 전략] .gitignore 격리 및 전송 브릿지 자동화
데이터 볼륨의 원천 격리: 프로젝트 초기 설계 단계부터 data/*.db 및 대형 로그 데이터를 .gitignore에 등록하여 깃허브 원격 저장소가 무거워지는 현상을 완전히 방지했습니다.

독립적 수동 이송 및 무결성 확보: 코랩 가속 환경에서 데이터 부재로 훈련이 정지되는 것을 막기 위해, 격리된 로컬/Codespace 내의 데이터 볼륨을 클라우드 전용 보안 브릿지망(Google Drive API 연동 혹은 Codespace 임시 대역폭 강제 업로드 git add -f)을 활용해 가상 세션의 data/ 경로로 정밀하게 수동 타겟 배포를 수행했습니다.

획득한 가치: 이 설계를 통해 향후 데이터셋이 수십 기가바이트(GB) 규모의 원시 데이터(Raw-tick Data)로 급격히 확장되더라도, 코드 버전 관리 시스템의 오염 없이 안정적으로 서빙 아키텍처를 독립 유지 및 서빙할 수 있는 MLOps 체력의 기반을 닦았습니다.

🚨 3. VRAM 하드웨어 디바이스 불일치 (Device Mismatch)
[문제 상황] 하드웨어 가속 연산 시 디바이스 불일치로 인한 구동 정지
코랩 GPU 세션 기동 시 RuntimeError: expected self and mask to be on the same device, but got mask on cpu and self on cuda:0 에러 발생으로 가속 엔진 정지.

[원인 분석 및 해결책]
원인: 모델 가중치와 입력 데이터 배치는 GPU 메모리로 정상 수송되었으나, 어텐션 연산 중간에 새롭게 동적 생성되는 Causal Mask 행렬(torch.ones)이 파이토치 기본 텐서 생성 규칙에 의해 CPU 단에 고착화되어 연산 충돌 발생.

해결책 (Dynamic Device Cloning): 마스크 텐서 선언 부에 하드웨어 타겟 주소를 고정하는 대신, 입력 임베딩 텐서의 디바이스 위치를 실시간으로 복사해 오는 device=embedded.device 속성을 강제 주입하여 해결 환경(CPU/GPU)에 관계없이 텐서들이 하나의 물리적 칠판 위에서 정렬되도록 동기화 완료.
